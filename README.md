# docker-php

Пробую собрать удобную среду для PHP разработки. Основные требования:

- Минимализм: ничего лишнего, только необходимое.
- Простая и понятная конфигурация
- По возможности, небольшой размер
- Сборка на основе официальных Docker образов.
  Если требуются какие-либо серьезные дополнения, устанавливаем их поверх оф. образов.

При сборке конфигураций использовал рекомендации AI: Grok, Gemini, Алиса.

## Вариант 1

Исходный вариант, который я сам использую при разработке большинства своих проектов.

**Достоинства**: малый размер контейнеров, нетребовательность к ресурсам.  

**Недостатки**: возможная проблема с функцией `glob()` в связи с отсутствием в Alpine Linux константы `GLOB_BRACE`.
В собственных разработках это не является серьезным ограничением, можно использовать альтернативы, однако если
приходится работать с готовыми проектами и отказаться от `GLOB_BRACE` невозможно, то это может являться ключевым
фактором и в этом случае следует использовать одну из альтернативных сборок среды разработки.

Контейнеры:

- **Alpine, Apache + SSL**  
  Собирается свой контейнер на основе образа `httpd:2.4-alpine3.22`.  
  Включены необходимые модули, настроен SSL.  
  Открыты порты 80 и 443.
- **Alpine, PHP 8.3 FPM**  
  Собирается свой контейнер на основе образа `php:8.3-fpm-alpine3.22`.   
  Включены наиболее часто используемые расширения.  
  Для иммитации `sendmail` используется `msmtp`.
- **Database**  
  MariaDB, используется оф. образ `mariadb:lts`.  
  При первичной инициализации контейнера есть возможность загрузить исходные данные. Для этого необходимо создать дамп
  своей базы данных (структура и данные)
- **Email Testing**  
  Для тестирования отправки Email сообщений используется сконфигурированный в PHP контейнере SMTP клиент `msmtp` и
  данный образ `mailhog/mailhog`.
